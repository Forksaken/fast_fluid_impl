# Оптимизации без многопоточности
- Использование спецификатора `inline` для части функций
- Изменение функции `get` в `VectorField`
Нечестным (но рабочим) способом оптимизации будет являться использовние флага O3 для компиляции программы.

## Другие возможные методы оптимизации
Данные методы не оказывают положительного влияния на работу программы на моей версии Windows, однако могут повысить скорость работы на других ОС и/или версиях Windows.
- Использование `printf` вместо `cout` для вывода данных массива
- Использование буфферизации вывода при помощи `std::ios_base::sync_with_stdio(false)` и `std::cin.tie(nullptr)`. Таким оброазом, можно отключить синхронизацию потоков вывода C++ и привязку `std::cin` и `std::cout`.
- Использование `int_fast32_t` вместо `int32_t`.
